Buenas prácticas de edición de código en JS

1- Realizar comentarios

Comentariar muchas veces puede parecer muy tedioso sobre todo cuando se hacen trabajos en un tiempo muy corto pero hacer los comentarios pueden ser de mucha ayuda, por si tiempo después necesitas revisar el código, debido a que esto te permitira saber que o para que funcionaba cierta parte del código, además esto facilita el proceso de modificacióno o revisión de tu código para otros programadores.

2- Longitud de caracteres

Es muy necesario recalcar el uso máximo de 80 caracteres por línea, debido a que muchas veces tendemos a escribir todo, o la mayor parte del código en una sola línea, generando así mucho desorden y una gran dificultad para la búsqueda de partes especificass del código en dado caso de que estas necesiten ser revisadas o modificadas.

3- Usar el punto y coma

Debes tener en cuenta que se puede escribir el código sin la necesidad de estar escribiendo el punto y coma al final de una instrucción, esto ya que Javascript utiliza ASI(Automatic Semicolon Insertion), pero a pesar de que esto sea muy válido al omento de encontrarnos con errores en el código esta falta de punto y coma nos dificultara en gran medida la identificación y corrección de los mismos, haciendo asi que perdamos mucho tiempo por no poner el punto y coma.

4- Declara mejor las variables

Aprender a declarar las variables, permitira tener un código más limpio y nos permite tenr un mejor conocimiento de nuestra variables. Sin embargo, te preguntarás de debes de hacer para tener una mejor declaración de variables, lo primero es declarar tu variables con const y let, evitando asi el uso de var, el cual declara las variables por nosotros. Otra recomendación para poder declarar variables de una mejor manera es hacerlo de manera local debido a que así sabremos de mejor manera en que ámbito es utilizada cada variable.

5- Scripts al final de la página

Colocar nuestros scrits antes del cierre de la etiqueta body nos permite que nuestras páginas carguen mucho más rápido para nuestros usuarios debido a que cuando un script se ejecuta en un navegador este no carga el resto de nuestra página hasta que no se haya terminado de cargar por completo el script, generando así una mala experiencia para el usuario.

6- Uso de === / !==

Esto debe de ser aplicado en el caso de querer comparar true o false es mucho más recomendado utilizar === debido que al trabajar con == y !=, muchas veces los valores de las variables que deseemos comparar pueden coaccionar generando así muchos problemas.

7- Uso de coma en lista de variables

Por lo general cuando escribes una lista de variables tiendes a escribir la palabra var antes de cada definición, pero lo mejor y más eficiente seria solo escribir una vez var y poner una coma por cada nueva variable que deseemos definir.

8- Usar Lint

Este tema se abarca en la siguiente página con más detenimiento, pero básicamente lo que hace es escanear nuestro JS en busca de errores de convenciones de estilos y de sintaxis, luego nos ayuda a reconocer dichos errores para poder corregirlos.

9- Funciones autoejecutabales

Estas nos permitiran hacer que nuestra función se ejecuten cuando se carga la página o si estas se ponen dentro de una función, ahorrandonos asi el proceso de llamar la función para que esta se ejecute.
